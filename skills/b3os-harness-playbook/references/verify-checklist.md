# harness 산출물 검증 체크리스트 (owner 필수)

병렬 agent 출력은 **그대로 채택 금지**. owner가 아래를 통과시킨 것만 채택.

## 1. 종합·정규화 (dedup)
- [ ] agent별 카운트/분류 **기준 통일** (call-site vs function 등 단위 불일치 제거)
- [ ] 경계 **중복 제거** (디렉토리 겹침 등으로 같은 항목 2번 잡힌 것)
- [ ] 누락 영역 없는지 — 각 agent 담당 범위가 전체를 덮었나

## 2. 환각 필터 (adversarial verify)
- [ ] **근거없는 정밀수치** 표시·제거 ("40~60% 절감", "$2-5/run" 류 — 출처 없으면 컷)
- [ ] 핵심 주장은 **반증 시도** — 살아남는 것만 채택
- [ ] 인용·파일경로·사례는 **실재 확인** (agent가 지어낼 수 있음)
- [ ] 추측은 **추측이라 명시** (확정처럼 쓰지 않기)

## 3. 채택 판정
- [ ] 위 통과분만 최종 산출물에 반영
- [ ] 불확실·미검증 항목은 "미검증"으로 라벨링하거나 제외
- [ ] 커버리지 줄였으면(top-N·샘플) **명시** (조용한 truncation 금지)

> 파일럿 교훈: 병렬은 중복+환각을 낸다. 이 패스를 건너뛰면 harness는 솔로보다 나쁘다.
