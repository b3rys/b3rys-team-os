03 — RESPONSE & COLLAB

응답 규칙 & 협업 구조

누가 언제 응답하는가. 기본은 mention-only, Codex만 default-intake. 순수 결정·승인·판단은 ask_gd(GD에게 위임).

DEFAULT MODE
mention-only
직접 멘션 · 답장 · sticky owner · direct inbox 일 때만 응답. 그 외엔 침묵.
"@스티브 이거 봐줘" — 멘션·답장·sticky·DM 시에만 owner
EXCEPTION
Codex = default-intake
미언급 메시지를 local LLM(로컬 판단 모델)으로 접수자(owner) 판단. 조율 성격 기본 담당.
"누구 멘션도 없는 요청" — Codex가 local LLM으로 owner 추론
INTAKE SCOPE
담당 영역 분리
Bill = infra_ops, Codex = general_pm, 나머지 = none. 순수 결정/승인/판단 → ask_gd.
infra 이슈 → Bill · PM 조율 → Codex
OWNER PRIORITY
explicit mention 우선
Sticky owner(직전 담당자) 유지. 단 본문 @멘션은 reply-to 담당자 우선권을 무시하고 가장 우선.
A 답장 + 본문 @B — owner는 B, A는 침묵
VISIBILITY · COORDINATOR
전체 맥락
Bill · Codex = 전체 맥락 관찰. 전 메시지 흐름을 보고 owner를 추론·조율한다.
coordinator — 배정 안 된 메시지도 관찰 가능
VISIBILITY · SPECIALIST
배정 + reply만
specialist는 배정된 메시지 + reply context(답장 맥락)만 본다. 확장은 coordinator/GD 승인.
Hermes — 자기 메시지만, 나머지는 비가시

05 — ONBOARDING

온보딩 프로세스 15단계

5 그룹 × 3 단계. 역할 정의부터 검증까지. 자격증명은 .env/credential store에만, TEAM-OS는 참조(복붙 금지).

1–2 · 정의
역할 정의 → 런타임 선택(claude_channel / openclaw / hermes_agent / b3os_native).
3–4 · 셋업
workspace · persona 파일 생성. TEAM-OS 참조(복붙 금지). 자격증명은 .env / credential store에만.
5–6 · 등록
텔레그램·Slack 연결 → agents.json 등록(id · display_name · role · runtime · status_provider · workspace · persona · telegram_bot_username).
7–9 · 라우팅
라우터 alias · bus wake allowlist · 그룹방 응답 정책(기본 mention-only).
10–12 · 멘션 게이트
멘션 패턴 매칭(공식 bot username + 팀 별칭) · self gate · reaction/ack gate.
13–15 · 검증
context visibility 정의 · 대시보드/문서 갱신 · 검증(one-shot → shadow test → bus DM smoke → 필요시 visible test).
DEVON 예시
id=devon · runtime=openclaw · role=스태프 엔지니어 · PM owner=Codex · ops reviewer=Bill · mention-only + default intake=none.
자격증명 원칙
평문 노출 금지. .env / credential store에만 저장하고 경로로만 참조. TEAM-OS 규칙은 복붙하지 않고 참조.
06 — VERIFICATION

온보딩 검증 기준

생성됨 ≠ 동작함. 라우팅·반응·가시성을 실제 표시 기준으로 통과해야 활성화.

라우터 테스트
멘션 · 답장 · sticky owner 정확도 확인.
Bus DM smoke test
메시지 버스 DM이 실제 도달·응답하는지 점검.
Team visible test
텔레그램 팀방에 실제로 표시되는지 확인. bot 초대 필수.
Reaction 검증
owner 메시지에만 반응. 비owner · 타팀원 명시멘션 · 예시영역 멘션엔 반응 없음.
멘션 패턴
@헤르메스 · @헤르 · @hermes 모두 작동(공식 username + 팀 별칭).
Context visibility
specialist는 배정 + reply만. coordinator 승인 시 전체 맥락.
EXPLICIT MENTION WINS
명시 멘션 우선
A에게 보낸 답장이어도 본문에 @B가 있으면 owner는 B. A는 침묵한다.
reply→A, 본문 @B — owner=B, A silence

Phase 2 — ACTIVATION
Devon 정식 온보딩
2026-06-01 dry-run / activation-ready. GD 승인 필수.
Phase 3 — ROLE CONTRACT
프로젝트-루프 역할 계약
Codex(PM · task 분해 · 단계 보고) · Devon(주 개발 실행) · Bill(ops/infra 검증) · Demis(search/AI/model 전문) · GD(결정권 · 승인).
Phase 4 — OFFBOARDING
오프보딩
