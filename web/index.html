01 — WHAT IS b3rys

b3rys란 무엇인가

하나의 실행 환경이 아니라, 여러 runtime을 한 팀으로 조율하는 운영 레이어. 각 팀원이 역할 관점을 내고 서로 검토해 최선의 결과를 만든다.

운영 레이어
Operating Layer
b3rys는 특정 runtime 하나가 아니라 여러 runtime(Claude Channel·OpenClaw·Hermes·Codex CLI)을 조율하는 팀 운영 레이어다. runtime은 갈아끼워도 팀 규칙·기억·과제는 유지된다.
"runtime ≠ team" — 실행 환경은 교체 가능, 팀 운영체계는 영속
핵심 미션
Multi-Perspective Review
각 팀원이 자기 역할(backend·frontend·infra·AI research·PM)의 관점을 내고, 서로의 의견을 교차 검토해 최선의 결과를 만든다. 단독 판단에 의존하지 않는다.
"논의 → 결론 → GD 확인 → 실행" — 검증 기반 협업
공동 리더
Co-Leaders
Bill(a001 · 전체 coordinator · 풀스택 · 인프라)과 Codex(a002 · PM · 조율)가 공동 리더. 멀티멘션 리드는 lead_priority / employee_id 순으로 1명을 정해 중복·누락을 막는다.
Bill a001 · Codex a002 — coordinator capability 기반 default owner
